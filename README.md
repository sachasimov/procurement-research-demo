# Procurement & Supply-Chain Research — Demo Pack

This repository shows the Linkup research endpoint applied to seven procurement and
supply-chain use cases. Each report is a single research job run at **reasoning depth L**
(`mode: investigate`, `outputType: sourcedAnswer`), saved verbatim with its query, run
time, answer, and full source list.

These use cases were chosen deliberately: they are questions whose ground truth is
**publicly documented and verifiable** (supplier capabilities, filings, regulations,
tariff schedules, published price indices, news). That is where web research is
strongest. Questions that depend on **negotiated, private, or tacit commercial data**
— for example the exact per-unit quote for a bespoke manufactured item at a given
volume — are a poor fit and are better handled by a request-for-quote process.

## How to read each report

Every report contains:

- **Why this is a good fit** — one line on why the question is web-grounded.
- **A parameters table** — reasoning depth, mode, run time, and number of sources.
- **The exact query sent** — so the prompt can be reviewed and reused.
- **The answer** — the model's synthesis, with inline citations.
- **Sources** — the full, de-duplicated list of sources the answer drew on.

## Reproducing

Each report is generated by [`generate.py`](./generate.py). With a Linkup API key in the
environment:

```bash
LINKUP_API_KEY=... python generate.py
```

Raw JSON responses for each job are stored in [`raw/`](./raw).
